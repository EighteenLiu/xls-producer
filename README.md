# Summary_xlsx_producer 月报工具

用于生成街道问题月报汇总表、补充汇总表中的部分统计项，以及生成 Word 工作报告。当前默认使用桌面窗口界面，不需要打开浏览器。

## 环境要求

- Windows
- Python 3.10 及以上
- Excel 文件处理依赖：

```powershell
pip install openpyxl python-docx pywin32
```

如果使用的是 Anaconda 环境，请先切换到项目所用环境后再安装依赖。

## 启动方式

在 PowerShell 中进入项目目录：

```powershell
cd D:\pycharm\Summary_xlsx_producer
python app.py
```

运行后会直接打开“月报工具”桌面窗口。

旧版 Web 页面入口仍保留，备用启动方式：

```powershell
python app.py --web
```

然后打开：

```text
http://127.0.0.1:8000
```

## 功能说明

### 生成汇总表

点击主窗口中的“生成汇总表”，会打开汇总表处理窗口，包含四个页签。

#### 生成月报汇总表

需要选择：

- 汇总 xlsx 模板
- 数据源台账
- 开始日期
- 结束日期
- 文件保存地址

点击“生成汇总表”后，程序会按照结束日期生成文件名，并保存到所选目录。

#### 更新容器总数与纯净率

需要选择：

- 带容器总数的原始台账，支持 `.xlsx` / `.xls`
- 已生成的月报汇总表
- 开始日期
- 结束日期
- 更新后汇总表保存地址

程序会生成容器总数与纯净率对照表，并更新汇总表中的对应行，其他内容保持不变。

#### 更新市级检查情况

需要选择：

- 已生成的月报汇总表
- 市级小区汇总表
- 市级社会汇总表
- 开始日期
- 结束日期
- 更新后汇总表保存地址

程序只更新汇总表中的“市级检查情况”相关行，其他内容保持不变。

#### 更新中转站情况

需要选择：
- 中转站台账，支持 `.xlsx` / `.xls`
- 已生成的月报汇总表
- 开始日期
- 结束日期
- 更新后汇总表保存地址

程序会按照所选时间段筛选中转站台账，并按街道更新汇总表中的“中转站”相关行，其他内容保持不变。

### 生成工作报告

点击主窗口中的“生成工作报告”。

需要选择：

- 各街道问题汇总表
- 源台账
- 垃圾分类检查自查报告模板
- 社会单位、餐饮单位检查报告模板
- 开始日期
- 结束日期
- 报告保存地址

点击“生成两个报告”后，会输出两个 Word 报告文档。

## 默认目录

程序会使用以下工作目录：

- 上传和临时文件：`web_work/uploads`
- 默认输出文件：`web_work/outputs`
- 中间处理文件：`web_work/process`

这些目录会在运行时自动创建。

## 使用注意

- 日期格式请使用 `YYYY-MM-DD`，例如 `2026-04-20`。
- 如果目标 Excel 或 Word 文件已经打开，程序可能无法覆盖，请先关闭同名文件。
- 模板和台账应保持工作表名称、街道列、日期列等结构稳定。
- 生成完成后，程序会尝试自动打开输出文件夹。

## 常见问题

### 运行后还是出现浏览器地址

说明你启动的是旧的 Web 服务，或还有旧终端没有关闭。先在旧终端按 `Ctrl+C` 停止，再运行：

```powershell
python app.py
```

### 点击按钮没有反应或报错

请查看 PowerShell 中的错误信息。常见原因是文件正在被 Excel 或 Word 打开，或者选择的文件不是对应模板/台账。

### 想使用 Web 版

使用：

```powershell
python app.py --web
```
